# Diocesan Finance Management System

A full-stack financial management system for Catholic dioceses to track parish remittances, collections, and outstanding balances.

## 🛠️ Tech Stack

### Backend
- **Runtime:** Node.js 18+ (ES Modules)
- **API:** Apollo Server 3 + GraphQL
- **Framework:** Express.js
- **Auth:** JWT (jsonwebtoken) + bcryptjs
- **Database:** PostgreSQL via `pg` connection pool
- **File Upload:** Multer + ExcelJS + csv-parse (XLSX/CSV)
- **Deployment:** Render (free tier)

### Frontend
- **Framework:** React 18 + Vite
- **GraphQL Client:** Apollo Client v4
- **Styling:** Tailwind CSS + shadcn/ui components
- **Charts:** Recharts
- **Icons:** Lucide React
- **Routing:** React Router v6
- **Deployment:** Vercel (free tier)

### Database
- **Provider:** Supabase (PostgreSQL 16)
- **Production Target:** AWS RDS PostgreSQL

## 📊 Database Schema

6 core tables:

| Table | Purpose |
|---|---|
| `users` | ADMIN, BISHOP, PRIEST accounts |
| `parishes` | Diocese, location, contact info |
| `collections` | Dynamic collection types (First Collection, Tithe, Harvest, etc.) |
| `remittance_records` | One record per parish per month |
| `remittance_line_items` | Amount per collection per record |
| `debtors` | Outstanding balance tracking per parish |
| `audit_logs` | Full JSONB before/after change tracking |

## 🚀 Local Development

### Prerequisites
- Node.js 18+
- PostgreSQL database (or Supabase free account)
- GitHub Codespaces (recommended) or local machine

### 1. Clone and install

```bash
git clone https://github.com/Mkjusti1/diocesan-finance-api
cd diocesan-finance-api
npm install
```

### 2. Configure environment

```bash
cp .env.example .env
```

Edit `.env`:

```env
NODE_ENV=development
PORT=4000
DB_HOST=your-db-host
DB_PORT=5432
DB_NAME=postgres
DB_USER=your-db-user
DB_PASSWORD=your-db-password
JWT_SECRET=your-super-secret-jwt-key
```

### 3. Run the database schema

```bash
# Using psql
psql -d your_database < schema.sql

# Or paste schema.sql into Supabase SQL Editor
```

### 4. Seed initial data

```bash
# Create the first ADMIN user
node seed-admin.mjs

# Seed parishes and collections
node seed.mjs
```

### 5. Start the backend

```bash
npm run dev
# API running at http://localhost:4000
# GraphQL playground at http://localhost:4000/graphql
```

### 6. Start the frontend

```bash
cd client
npm install
npm run dev
# Frontend running at http://localhost:5173
```

The Vite dev server proxies `/graphql` and `/api` requests to `http://localhost:4000` automatically.

## 📤 Bulk Upload Format (XLSX)

Upload `.xlsx` files with one sheet per month, named `JANUARY`, `FEBRUARY`, etc.

| Parish Name | First Collection | Second Collection | Tithe | Harvest | Special Offering | Donations | Other Income |
|---|---|---|---|---|---|---|---|
| St. Peter's Parish | 120000 | 80000 | 150000 | 0 | 50000 | 0 | 0 |
| St. Paul's Parish | 95000 | 60000 | 120000 | 0 | 0 | 30000 | 0 |

- Duplicate records (same parish + month + year) are automatically skipped
- Parish names must match exactly as stored in the database
- Upload endpoint: `POST /api/upload` with `file`, `year` form fields

## 🔐 API Authentication

All GraphQL queries and mutations require a Bearer token in the Authorization header:

```
Authorization: Bearer <jwt_token>
```

**Get a token:**

```graphql
mutation {
  login(input: { email: "admin@diocese.com", password: "yourpassword" }) {
    token
    user { id name role }
  }
}
```

**Priest token login:**

```graphql
mutation {
  loginWithToken(token: "priest-one-time-token") {
    token
    user { id name role parishId }
  }
}
```

## 📡 Key GraphQL Operations

### Queries

```graphql
# Dashboard statistics
dashboardStats(year: 2026) { totalCollectedThisYear totalParishes totalOutstanding }

# Monthly breakdown
monthlySummary(year: 2026) { monthName totalCollected parishCount }

# Parish summaries
parishSummaries(year: 2026) { parish { name } totalCollected monthsReported outstandingBalance }

# Remittance records with filters
remittanceRecords(year: 2026, month: 1, parishId: "1") { id monthName totalAmount parish { name } }

# Priest: own parish only
myParishRemittances(year: 2026) { id monthName totalAmount }
```

### Mutations

```graphql
# Create a parish
createParish(input: { name: "St. Mary's", diocese: "Lagos", location: "Ikeja" }) { id }

# Record remittance manually
createRemittanceRecord(input: {
  parishId: "1", year: 2026, month: 4
  lineItems: [{ remittanceSourceId: "1", amount: 150000 }]
}) { id totalAmount }

# Create a priest user (returns one-time token)
createUser(input: { name: "Fr. John", role: PRIEST, parishId: "1" }) { id priestToken }
```

## 🚢 Deployment

### Backend (Render)

1. Connect GitHub repo to Render
2. Set environment variables in Render dashboard
3. Build command: `npm install`
4. Start command: `npm start`
5. Auto-deploys on every push to `main`

### Frontend (Vercel)

1. Import repo to Vercel
2. Set Root Directory to `client`
3. Add environment variable: `VITE_API_URL=https://your-api.onrender.com`
4. Auto-deploys on every push to `main`
